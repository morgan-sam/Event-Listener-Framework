ELF - Event Listener Framework

ELF (event listener framework) is a library to simplify the process of adding and removing event listeners.

ELF uses haskeys to identify event listeners so specific event listeners can be easily removed using a key. In vanilla JS the exact arguments used on the add call must match the remove call, which can lead to confusion if the specified arguments have changed value since the event listener was created. ELF stores the add arguments in an array so they can be easily accessed when they are needed to remove an event listener

ELF allows for removing multiple event listeners at the same time, either via element, the entire document, or using categories (which work similarly to CSS classes).

ELF adds the following new functions:

createEventListener(elementID, eventType, elementFunc, eventCategory); Similar to addeventlistener, except the event target must be provided by ID. Also allows for categories to be specified.
element.deleteEventListener(hashkey); Removes a specific event listener from an element using a specified hashkey.
element.deleteAllEventListeners(); Removes all event listeners from the element is it called on.
deleteEventListenerByCategory(categoryInput); Removes all event listeners of specified categories, or multiple category combinations. Spaces and dots work as logical OR and AND operators respectively, just like CSS classes.
deleteDocumentEventListeners(); Removes all event listeners on document.

How to use the demo and tutorial

The demo and tutorial provided help to make clear the functionality and usage of ELF.

Demo

The demo section shows the basic functionality of ELF without showing the code. It has a button for creating event listeners, and buttons for the 4 methods of deleting event listeners.

The pop up prompts will ask for the revelant information needed to create/delete the event listeners (i.e. which button to assign, what message to print, and what categories to assign).

Assigned messages will print on screen in the message box, and created event listeners will appear in the active event listeners box.

Try making event listeners with as many different parameters as you can so you can try out each different way of deleting event listeners in ELF.

Remember multiple categories can be specified at once; with a period ( . ) specifying an AND operator and a space (   ) specifying an OR operator.

Tutorial

The tutorial section runs through the code of each different command via an ordered set of buttons with the code text displayed.

Each step has a numerically labelled button which adds code to the 'Code Input' section at the top. Once all steps of a given tutorial are completed, press 'Run Code Input' to run the full command.

For adding event listeners, each step will give multiple options for creating event listeners. As with the Demo try as many combinations as you can to see how each command is used programatically.

For removing event listeners, select a tutorial from the dropdown list of each 4 methods. Try all 4 to understand the distinct advantages of each method.

For the delete by hash tutorial, the list of active hashkeys will appear as a dropdown list.

For the delete by category tutorial, a table of each active category will appear. Select the corresponding remove checkbox to remove a category.

Categories can be grouped together by selecting remove and any of the 3 groups. Try several different grouping options and click assign without running code to see how the groups affect the code input text.

Example Use Case

You happen to be currently looking at an example of where ELF has proved advantageous! In creating this guide for the extension I came across an issue. Event listeners created on the Demo screen also appeared on the Tutorial screen, which caused an interface attempting to simplify the explanation to become overly complicated. Removing a category on one screen would also affect any categories of the same name on the other. I wanted both the Demo and Tutorial screens to have Event Listeners and controls that were completely independent of each other.

With the categories functionality I was able to achieve exactly that. Every EL created in the Demo screen has a hidden 'demo' category, and every EL created in the Tutorial screen has a hidden 'tutorial' category. On deleting ELs by category on either screen, for each category input either '.demo' or '.tutorial' is added (i.e. 'msg new' becomes 'msg.demo new.demo'). This will only remove specified categories in the appropriate screen, giving the exacly functionality I needed for this guide.
